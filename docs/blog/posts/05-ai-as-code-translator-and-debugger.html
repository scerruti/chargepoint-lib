Once the architecture was chosen (Cloudflare Workers), the challenge shifted to implementation details. I leaned on AI as a translator between the Python-centric community docs and the TypeScript/Workers runtime, and as a debugger for all the sharp edges hidden in HTTP minutiae.

From Python Examples to TypeScript Workers

Community examples are mostly Python. Workers run in a V8 isolate with a Web-standard API surface‚Äîno requests, no fs, no Node Buffer. AI helped map common patterns:

  • requests.post ‚Üí fetch: JSON body for modern endpoints; URL-encoded form for legacy NA endpoints.
  • requests.Session ‚Üí manual cookie header: Capture Set-Cookie and forward it explicitly in the Cookie header.
  • Python datetime ‚Üí Temporal / Intl in Workers: Check America/Los_Angeles hour before acting.

Debugging the Sharp Edges (with AI in the loop)

  1. User-Agent enforcement: ChargePoint/6.113.0 (com.chargepoint.mobile; build:1; iOS 16.0.0) is required. Without it, the API returns HTML 404 or 403.
  2. Legacy NA shard: Modern docs say account.chargepoint.com; my account is on na.chargepoint.com. Different endpoints, form-encoding instead of JSON.
  3. Password encoding: # must be percent-escaped (%23) in form data. The fix was to use URLSearchParams or --data-urlencode in curl.
  4. Station ID format: Needs the 1: prefix (e.g., 1:214541061163). Raw serial numbers fail.
  5. WAF behaviors: ChargePoint sometimes returns HTML for errors. AI suggested logging the entire body on failure to catch hidden messages.

Each of these was solved by pairing with AI: describe the symptom, paste the response, get a hypothesis, and test the fix. The loop time was minutes instead of hours.

Building the Worker Skeleton

export interface Env {
  CP_USERNAME: string;
  CP_PASSWORD: string;
  CP_STATION_ID: string;
  ALERT_EMAIL: string;
  TZ_REGION: "America/Los_Angeles";
}

export default {
  async scheduled(event, env, ctx) {
    const now = new Date();
    const hour = new Intl.DateTimeFormat("en-US", { timeZone: env.TZ_REGION, hour: "numeric", hour12: false }).format(now);
    if (Number(hour) !== 6) return;

    ctx.waitUntil(runStartCharge(env));
  }
};

AI helped keep this tight and compliant with Workers constraints‚Äîno extraneous dependencies, all Web APIs.

Retries and Alerting

Another pattern I asked AI to draft: a retry wrapper with exponential backoff (2s, 4s, 8s) around both login and start_session calls, plus a MailChannels alert on final failure.

The biggest gain from AI wasn't generating boilerplate; it was shortening the diagnosis loop when an HTTP quirk or platform constraint surfaced.

Outcome

Weeks of reverse-engineering condensed into a day of paired debugging. The Worker now logs clearly, retries sensibly, and sends alerts when things go wrong‚Äîall while respecting the charger‚Äôs native scheduling.
